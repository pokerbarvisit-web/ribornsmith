# Drive連携セットアップ（Google Apps Script）

キャラ箱庭（`character_diorama.html`）に Google Drive のフォルダから
キャラ画像を自動で流し込むための手順です。所要 約10分。

## 仕組み

```
[画像生成] → [Driveの共有フォルダに保存]
                     │
        GAS ウェブアプリ（doGet）が
        ① キャラ一覧を JSON で返す（…/exec）
        ② 画像本体を base64 で返す（…/exec?img=ID）
                     │
   キャラ箱庭が 15秒ごとに①を fetch → 新規キャラだけ追加
   画像は②経由で取得するので、白背景の生成画像も
   自動で白抜きされ、キャラだけが地面に立つ
   （31体目以降は古い順に消え、常に最新30体を表示）
```

②が使えない場合は自動でDriveサムネイルに切り替わります
（その画像だけ白抜きなしの「アクスタ風」枠になります）。

## 手順

### 1. Drive にキャラ用フォルダを作る

1. Google Drive で新しいフォルダを作成（例: `キャラ箱庭`）。
2. フォルダを開いた状態の URL からフォルダIDを控える。
   `https://drive.google.com/drive/folders/【ここがフォルダID】`
3. フォルダの共有設定を「**リンクを知っている全員（閲覧者）**」にする。
   ※サムネイルURLを匿名で表示するために必要です。

### 2. GAS プロジェクトを作る

> **今回のイベント用**: フォルダ「ご当地キャラを作ろう！」
> （ID: `1KoexIA7NFdKlvBOOSu5aVyOwQoINUqqV`）を設定済みの完成コードが
> 同じフォルダの **`code.gs`** にあります。中身をそのままコピーすればOKです。

1. <https://script.google.com> → 「新しいプロジェクト」（フォルダの所有者
   `pokerbar.visit@gmail.com` のアカウントで作成する）。
2. エディタの `コード.gs` の中身を、本フォルダの **`code.gs`** の内容で**全置換**する。

キャラ名はファイル名から自動で切り出します:
`地区_キャラ名_日付_時刻.png` → 名札「キャラ名」。
キャラ名が「なまえなし」のときは地区名を使います（例:
`平田_平田ちゃん_20260708_145757.png` → 「平田ちゃん」、
`錦・深川_なまえなし_20260708_151704.png` → 「錦・深川」）。

### 3. ウェブアプリとしてデプロイ

1. 右上「デプロイ」→「新しいデプロイ」。
2. 種類の選択（歯車）→「**ウェブアプリ**」。
3. 設定:
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員（匿名ユーザーを含む）**
4. 「デプロイ」→ 初回はアクセス承認（自分のアカウントで許可）。
5. 表示された **ウェブアプリURL（`…/exec` で終わるもの）** をコピー。

### 4. キャラ箱庭側の設定

手動で設定する場合:

1. `character_diorama.html` を開く。
2. 画面下の操作パネル「Drive」欄に手順3のURLを貼り付け → 「**読込**」。
3. 「**自動更新15秒**」にチェックを入れると、以降15秒ごとに
   新しい画像だけが自動で追加されます（会場サイネージ向け）。

**自動接続（サイネージ向けおすすめ）**: URLの後ろに `?gas=GASのURL` を
付けて開くと、起動と同時にDrive連携＋自動更新が始まります。
大画面ランチャーでも同様に引き継がれます。

```
bigscreen.html?gas=https://script.google.com/macros/s/XXXX/exec
```

### 動作確認

- Driveフォルダに画像を1枚入れる → 15秒以内に画面へキャラが登場すればOK。
- ブラウザで GAS の URL を直接開くと JSON の配列が見えるはずです。
  `[{"name":"…","url":"https://drive.google.com/thumbnail?id=…","id":"…","time":…}]`

## 生成PCからの自動アップロード（uploader.html）

各生成PCのローカルフォルダに保存されたキャラ画像を、自動でDriveへ送る
「見張り番」ページです。インストール・Googleログインは不要です。

1. 生成PCの **Chrome または Edge** で
   `https://pokerbarvisit-web.github.io/ribornsmith/uploader.html` を開く
2. 「📁 フォルダを選ぶ」→ キャラ画像が保存されるフォルダを選択
3. あとは開いたままにするだけ。5秒ごとに新しい画像を見つけて自動送信します
   - 同じファイル名は二重送信されません（PCを再起動してやり直しても安全）
   - 書き込み途中のファイルは完了を待ってから送ります
   - **ウィンドウは最小化しない**こと（裏に回すと送信間隔が約1分に伸びます）

流れ: 生成PCのフォルダ → (uploader) → Drive → (15秒毎の自動更新) → 箱庭に登場

※この機能には GAS の**再デプロイ**（`code.gs` を最新にして「新バージョン」で更新）が必要です。

## 運用のコツ・注意点

- **透過について**: 画像はGAS経由（`?img=ID`）で取得するため、白背景の
  生成画像も自動で白抜きされます（実イベント画像でチューニング済み:
  クリーム色のもや・足元の薄い影も除去）。GAS取得に失敗した画像だけ
  サムネイル表示＋「アクスタ風（白フチ枠）」にフォールバックします。
- **コードを修正したら**: GAS は「デプロイ」→「デプロイを管理」→ 鉛筆アイコン →
  バージョン「新バージョン」で更新しないと反映されません。
- **上限**: 表示は常に最新30体。31体目以降が来ると古いものから消えます。
- **並び順**: GAS が作成日時の古い順で返し、アプリは受信順（=生成順）に並べます。
- **URLの秘匿**: URLを知っていれば誰でも一覧JSONを取得できます。
  イベント用の専用フォルダを使い、イベント後はデプロイをアーカイブしてください。
